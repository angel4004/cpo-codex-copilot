# Границы artifacts

Называй границы artifact явно:

- draft;
- review;
- decision record;
- project passport;
- evidence gap report;
- PAF consistency review.

Не смешивай private raw context с shared artifacts.

## Проверка интерфейса перед handoff

Внутренний alias для проверок: `UI-evidence gate`.

Перед developer handoff, где есть интерфейс, фильтры, сценарии, навигация или
состояния экрана, явно проверь источник UI-evidence:

- актуальный интерфейс проверен через скриншоты/live read-only просмотр;
- актуальный интерфейс не проверен;
- есть только старый прототип, переписка или гипотеза.

Если актуальный интерфейс не проверен, не называй handoff готовым для
разработки. Назови его `черновик для проверки UI`, перечисли неподтвержденные
элементы и дай один следующий шаг проверки.

Карта сценариев для руководителя должна начинаться с полной рамки:
`хорошо / плохо / можно улучшить`. Top-5 для интерфейса выбирай после полной
карты, а не вместо нее.
