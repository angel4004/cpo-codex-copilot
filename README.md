# CPO Copilot для Codex

CPO Copilot работает по PAF. PAF - это продуктовая методология и
структурная линза для принятия решений: она помогает связать контекст продукта,
пользователя, потребность, ценность, evidence, риски, ограничения и следующий
практический шаг.

Автор PAF - Сергей Тихомиров. Официальное руководство:
[Product Architecture Framework](https://productframework.ru/ops/main).

PAF не заменяет суждение CPO и не является автоматической scoring machine. Если
evidence слабая, Copilot должен показать, чего не хватает, какие выводы пока
нельзя делать и что нужно проверить дальше.

## Для чего он нужен

CPO Copilot помогает фаундеру, CPO или продакту связывать идеи, фичи и решения
с целями продукта и бизнеса.

Он нужен, когда нужно принять или объяснить продуктовое решение: что проверять,
что менять, что отложить, что остановить, а что уже можно передавать в
разработку.

Перед решением CPO Copilot помогает собрать контекст:

- что уже известно;
- чего не хватает;
- кто пользователь;
- какую потребность закрываем;
- какую ценность создаем;
- на каких данных основаны выводы;
- что пока неизвестно;
- какие есть риски;
- какие варианты решения доступны;
- какой следующий шаг выбрать.

Контекст нужен не ради документа. Он помогает не защищать идею интуицией, а
показать ход решения: какую потребность пользователя закрываем, какую ценность
создаем, что уже проверено и какой риск остается.

Так команде проще договориться, а бизнесу - не тратить разработку на фичи без
понятного основания.

## Какие задачи он помогает решать

CPO Copilot стоит использовать не ради отдельных артефактов, а когда нужно:

- разобраться, стоит ли двигать продуктовую идею дальше;
- выбрать практический следующий шаг: проверить, сузить, изменить, отложить,
  остановить или передать в разработку;
- превратить разрозненные заметки, созвоны, метрики, гипотезы и stakeholder
  requests в понятную картину решения;
- подготовиться к разговору с фаундером, CPO, командой или бизнесом о том,
  почему выбран именно этот продуктовый шаг;
- увидеть, где решение держится на фактах, а где на предположениях;
- найти слабые места до того, как команда потратит разработку;
- понять, какие заявления про пользователя, ценность, PMF, PCF или эффект пока
  нельзя делать;
- выбрать, какие evidence нужно собрать дальше.

## Персональная польза для пользователя

CPO Copilot помогает снизить хаос вокруг продуктового решения:

- не держать всю логику в голове;
- не спорить с командой на уровне вкуса и интуиции;
- быстрее показать, что известно, что неизвестно и почему решение разумное;
- не обещать PMF, PCF, рост или ценность без evidence;
- сохранить ход рассуждения так, чтобы к нему можно было вернуться позже;
- получить рабочий черновик решения, а не пустой шаблон.

## Что это в версии Codex

CPO Copilot для Codex - это локальный Codex workspace, который превращает
обычный чат Codex в продуктового помощника с правилами, памятью, workflow и
проверками качества.

Отдельный сервер, GPT Project или OpenAI API key для обычного использования не
нужны. Copilot запускается как локальная папка, которую пользователь открывает в
Codex.

Git остается источником истины: когда владелец Copilot обновляет инструкции,
практики или проверки, пользователь может получить обновления через `git pull`.

## Что получится на выходе

В зависимости от задачи Copilot помогает получить:

- картину текущего продуктового решения;
- рабочий черновик project passport или его части;
- список недостающих evidence и открытых вопросов;
- список рисков и слабых мест;
- forbidden claims - что пока нельзя утверждать;
- рекомендации по следующему практическому шагу;
- PAF consistency notes, если нужно проверить соответствие методологии.

## Что нужно перед установкой

- установленный Git;
- установленный Codex;
- доступ к GitHub repo `angel4004/cpo-codex-copilot`;
- Windows и PowerShell для команд ниже.

Подробная инструкция по установке, обновлению и локальной памяти:
[docs/install.md](docs/install.md).

## Как установить

```powershell
git clone https://github.com/angel4004/cpo-codex-copilot.git cpo-codex-copilot
cd cpo-codex-copilot
powershell -NoProfile -File tools/setup-local-workspace.ps1
```

После успешного setup открой именно папку `cpo-codex-copilot` как workspace в
Codex и напиши:

```text
Активируй CPO Copilot
```

## С чего начать в чате

Начинай от работы, которую нужно сделать, а не от внутреннего режима.
Выбери ближайший вариант или напиши своими словами.

```text
Запустить новый продукт или направление: помоги сузить идею и выбрать первый проверочный шаг.
Развивать текущий продукт: помоги понять, что улучшать, запускать или отложить.
Подготовить продуктовый шаг: проверь фичу по ценности, evidence и рискам.
Разобрать спорный вывод: что можно утверждать и что нужно проверить дальше.
```

Например:

```text
Развивать текущий продукт: у нас есть несколько идей улучшений. Помоги понять,
что запускать, что отложить и какой evidence нужен для решения.
```

Дальше можно писать одну из практических формулировок:

```text
У меня есть идея фичи. Помоги понять, стоит ли ее проверять, менять, отложить
или передавать в разработку.
```

```text
Вот контекст продукта. Помоги собрать картину решения: что известно, чего не
хватает, какие риски и какой следующий шаг.
```

```text
Мне нужно объяснить команде, почему мы выбираем этот продуктовый шаг. Помоги
собрать аргументацию без неподтвержденных заявлений.
```

```text
Проверь мои выводы: где есть evidence, где предположения, а что пока нельзя
утверждать.
```

```text
Помоги подготовиться к разговору с фаундером или CPO о приоритете, риске и
следующем шаге.
```

## Как обновлять Copilot

Когда в Git появилась новая версия:

```powershell
git pull --ff-only
powershell -NoProfile -File tools/setup-local-workspace.ps1
```

Эта команда подтянет новые инструкции, practices, workflows, checks и shared
memory. Локальная память пользователя в `memory/local/` не должна
перезаписаться.

## Локальная память

Setup создает локальную память в `memory/local/`. Она нужна, чтобы Copilot мог
учитывать контекст пользователя и проекта между задачами.

Эта память ignored by git: она не должна попадать в commit или push. Если файл
локальной памяти уже существует, setup его не перезаписывает.

## Ограничения и безопасность

- Не храни API keys, пароли, session tokens, `.env*` или другие secrets в
  `memory/local/`.
- Не коммить `memory/local/`, `traces/local/`, `.env*` и raw private материалы.
- Перед push проверяй `git status --short --branch` и diff.
- Важные выводы по PAF, PMF, PCF, ценности или бизнес-эффекту требуют evidence
  refs и human review.
- Внешние отправки, dependency install, push, PR, deploy, provider/API calls вне
  Codex runtime, scheduler changes и записи в соседние проекты требуют явного
  подтверждения человека.

## Текущий статус

Эту версию можно использовать для controlled local testing и dogfood внутри
Codex. Это еще не production-certified Copilot: full production pass требует
отдельной проверки trusted Codex hooks и live behavior/protocol verification.
